AOP有哪些实现方式

AOP有两种实现方式：静态代理和动态代理

  1. 静态代理：代理类在编译阶段生成，在编译阶段将通知织入java字节码中，也称编译时增强，Aspectj使用的就是静态代理。代理对象需要与目标对象实现一样的接口，并且实现接口的方法，会有冗余代码，同时，一旦接口增加方法，目标对象与代码对象都要维护。

  2. 动态代理：代理类在程序运行时创建，AOP框架不会去修改字节码，而是在内存中临时生成一个代理对象，在运行期间对业务方法进行增加，不会生成新类### jdk动态代理和CGLIB动态代理的区别

  3. jdk动态代理，如果目标类实现类接口，Spring AOP会先择使用jdk动态代理目标类，代理类实现目标类实现的 接口动态生成，不需要自己编写，生成的动态代理类和目标类都实现相同的接口。代理类实现InvocationHandler接口，实现invoke()

  4. CGLIB动态代理，如果目标类没有实现接口，那么Spring AOP会选择使用CGLIB来动态实现代理目标类。在CGLIB可以在运行时动态生成类的字节码，动态创建目标类的子类对象，在子类对象中增强目标类。CGLIB是通过继承的方式做的动态代理，因此如果某个类被标记为final，那么它是无法使用CGLIB做动态代理的### AOP中的专业术语

  5. 连接点：目标类中可以被增强功能的方法

  6. 切点：目标类中被增强功能的方法

  7. 通知：又被称为增强，比如想给方法增加日志功能，那么这个功能就被称为 通知/增强

  8. 切面：通知/增强+切点=切面

  9. 目标：被增强的类的对象

  10. 织入：增强应用到目标的过程

  11. 代理：向目标对象应用通知后创建的对象### Spring通知有哪些类型

  12. 有5种类型：

  13. 前置通知（Before），在目标方法被调用之前调用通知功能

  14. 后置通知（After)，在目标方法完成之后调用通知，此时不会关心方法的输出是什么

  15. 返回通知（After-returning），在目标方法成功执行之后调用通知

  16. 异常通知（After-throwing），在目标方法抛出异常后调用通知

  17. 环绕通知（Around）,通知包裹了被通知的方法，在被通知的方法调用之前和调用之后执行自定义的逻辑### 什么是IOC，有什么好处

  18. 控制反转，由Spring容器管理bean的整个生命周期，通过反射实现对其他对象的控制，包括初始化、创建、销毁等，解放手动创建对象的过程，同时降低类之间的耦合度

  19. 降低了类之间的耦合，对象创建和初始化给了Spring容器管理，在需要的时候只需向容器进行申请### 什么是依赖注入

  7. spring扫描xml或者注解生成BeanDefinition，存入BeanDefinitionMap中

  8. BeanFactory，是负责创建bean的简单工厂，遍历BeanDefinitionMap，调用getBean方法，创建bean。在创建的过程中会先去一级缓存singletonObjects（Map<beanName,bean实例>）中看没有创建好的bean，如果有直接取，如果没有且不在创建中，直接返回null，如果在创建中，尝试从二级缓存中获取，二级缓存没有会得到一级缓存的锁，再重新尝试一级缓存、二级缓存、三级缓存获取，如果都没有就会返回null。

  9. 如果没有创建，首先标记这个bean正在创建中，然后会调用doCreateBean()来创建Bean，首先会实例化，在beanFactory中存放一个getEarlybeanReference函数，提前暴露bean，开始属性注入，这时发现A需要注入B，所以去getBean(B)，此时又会走一遍上面描述的逻辑，到了B的属性注入这一步。此时B调用getBean(A),这时一级缓存中找不到，但是发现A正在创建中，于是去二级缓存中查找，还是找不到于是去三级缓存中找，找到后B进行属性注入，aop生成代理类后，加入二级缓存，从三级缓存中移除bean，B初始化后进入一级缓存，A属性注入的

  10. 创建的时候分为三部分，实例化、属性注入、初始化，在实例化之后就将bean放入三级缓存BeanFactories，

  11. 一级缓存的目的是解决循环依赖时的死循环问题

  12. 二级缓存的目的是解决对缓存加锁时，性能查

  13. 三级缓存的目的时为了解决bean有动态代理，且要向不止一个类里依赖注入

  14. 非单例模式循环依赖和构造器注入的循环依赖问题，Spring无法解决

  15. 单例模式下的循环单例问题可以使用三级缓存来处理

  16. 单例对象实例化完成后加入singletonFactories，完成实例化未完成初始化，将对象移入到earlySingleObject，完成了初始化，将对象移入singleObject

  17. 属性注入的循环依赖主要通过将实例化完成的bean添加到singletonFactories来实现的，而使用构造器依赖注入的bean在实例化的时候会进行依赖注入，不会被添加到singletonFactories中### Spring的单例Bean是否有线程安全问题

Spring核心模块

Spring有七个核心模块

  1. Spring Core核心容器，核心容器提供Spring框架的基本功能，Spring以bean的方式组织和管理Java应用中的各个组件及其关系。Spring使用BeanFactory来产生和管理Bean，它是工厂模式的实现。BeanFactory使用控制反转模式将应用的配置和依赖性规范与实际的应用程序代码分开。BeanFactory使用依赖注入的方式提供给组件依赖，主要实现控制反转IOC和依赖注入DI、Bean配置以及加载。
  2. Spring AOP，spring面向切面编程，通过配置管理特性，Spring AOP模块直接将面向切面的编程功能集成到Spring框架中，所以很容易的使Spring框架管理的任何对象支持AOP。Spring AOp模块为基于Spring的应用程序中的对象提供了事务管理服务。通过使用Spring Aop，不用依赖EJB组件，就可以将声明性事务管理集成到应用程序中，Aop把一个业务流程分为几部分，例如权限检查，业务处理，日志记录，每个部分单独处理，然后把它们组装成完整的业务流程，每个部分被称为切面或关注点。Aop的实现原理为动态代理技术，一共有两种代理模式
  3. Spring context，Spring上下文。Spring 上下文是一共配置文件，向Spring框架提供上下文信息，Spring上下文包含企业服务，如JNDI，EJB，电子邮件，国际化，校验和调度功能，提供框架式Bean访问方式，其他程序可以通过Context访问Spring的Bean资源
  4. Spring Dao，此模块主要目的是将持久层相关问题与一般的业务规则和工作隔离开来，Spring中的Dao提供一致的方式访问数据库，不管采用何种持久化技术，Spring都提供一致的编程模式，Spring还对不同的持久层提供一致的Dao方式的，Spring的Dao模块对JDBC进行了再封装，隐藏了Connection，Statement，ResultSet等JDBC API ,使Dao模块直接集成jdbcDaoSupport类
  5. Spring ORM，spring与所有的主要的ORM框架都集成的很好
  6. Spring Web模块建立在应用程序上下文模块之上，为基于web的应用程序提供了上下文，web层使用web层框架，可选的，可以是Spring自己的MVC框架，或者提供了web框架
  7. Spring MVC，这是一个全功能的构造web应用程序的MVC实现，通过策略接口，mvc框架变为高度可配置的，Spring的MVC框架提供清晰的角色划分，控制器，验证器、命令对象、表单对象和模型对象、分发器、处理器和视图解析器。Spring支持多种视图技术。### MVC模型
    MVC是model、view和controller的缩写，分别代表web应用程序中的三种职责
  • model模型，用于存储数据以及能处理用户请求的业务逻辑
  • view模型，向控制器提交数据，显示模型中的数据
  • controller控制器，根据视图提出的请求，判断将请求和数据交给哪个模型处理，处理后的有关结果交给哪个视图更新显示### SpringMVC的工作原理
  1. 用户发送请求至前端控制器DispatcherServlet
  2. DispatcherServlet收到请求调用HandlerMapping处理器映射器
  3. 处理器映射器找到处理request的handler和interceptor，构造成HandlerExecutionChain执行链返回给DispatcherServlet
  4. DispatcherServlet调用HandlerAdapter处理器适配器
  5. 处理器适配器handlerAdapter经过适配调用具体的处理器Controller
  6. Controller执行完成返回ModelAndView
  7. HandlerAdapter将controller执行结果ModelAndView返回给DispatcherServlet
  8. DispatcherServlet将ModelAndView传给ViewReslover视图解析器
  9. ViewReslove解析后返回具体View
  10. DispatcherServlet将模型数据填充到视图View中
  11. DispatcherServlet响应用户### 什么是SpringBoot
    SpringBoot是Spring开源组织下的子项目，是Spring组件一站式解决方案，主要是简化了使用Spring的难度，简化了繁重的配置，提供了各种启动器，开发者能够快速上手。

Springboot的优点

  1. 内置servlet容器，不需要在服务端部署tomcat。只需要将项目打成jar包，使用java -jar xxx.jar一键启动项目
  2. Springboot提供了starter，把常用库聚合在一起，简单复杂的环境配置，快速搭建spring应用环境，可以避免大量的Maven导入和减少各种组件版本冲突发生可能性
  3. 可以快速创建独立运行的spring项目，集成各个主流框架
  4. 准生产环境的运行应用监控### Springboot的自动配置原理
