安装Ollama

启动qwen3.5官方模型

通过Modelfile启动模型

Modelfile就像Dockerfile一样，本质上是一个文本文件。Modelfile中常见的指令如下

指令 描述
FROM（必需） 定义要使用的基线模型。
PARAMETER 设置Ollama运行模型时的参数。
TEMPLATE 要发送给模型的完整提示模板。
SYSTEM 指定将在模板中设置的系统消息。
ADAPTER 定义要应用于模型的（Q）LoRA适配器。
LICENSE 指定法律许可。
MESSAGE 指定消息历史。

from指令用于指定基线模型，可以是ollama官网的现有模型，也可以是Safetensors 模型，也可以是GGUF 文件构建

大模型常见精度

  1. FP32：单精度浮点数，32位浮点数，最高精度
  2. FP16：半精度浮点数，16位浮点数
  3. BF16：16位格式，动态范围与FP32接近
  4. 混合精度：结合FP16/FP32，用FP16加速计算，用FP32保存关键参数（如权重、梯度），通常配合NVIDIA的AMP（自动混合精度）使用
  5. INT8：8位整数量化，将模型权重和激活值量化位8位整数，卓著减少内存占用和计算成本，但可能损失精度
  6. INT4：4位整数量化，更激进的量化方式，模型体积进一步缩小，但对精度影响较大，常用于边缘设备推理
  7. GPTQ：基于梯度的后训练量化方法，支持4/3/2位量化
  8. AWQ：通过保留关键权重的高精度提升效果
  9. GGUF：Ollama使用的量化格式，如Q4_K_V
